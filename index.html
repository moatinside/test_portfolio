— 02 // FRAMEWORK

THINKING
FRAMEWORK

01

ペインがないとビジネスはコケる

真のニーズは「あったらいいな」ではなく「解決しないと困る」の中にある。 行動変容コストの壁を乗り越えるペインの深さこそが、 ビジネスの起点。表面的な「技術進化」論ではなく、 構造的変化を設計する。

02

リアル1回 ＞ 仮想100回

机上の分析より、1回の面談。1回の現場。 仮説はドキュメントで検証するのではなく、 相手との対話の中で叩きつけて反応を見る。 「やった気になる」ことを徹底的に排除する。

03

一気通貫 — 評価→実行→再評価

YC→Alchemist→a16z由来の評価フレームワークで 仮説を検証。Go/No-Go/Archiveの判断で リソースを最適配分する。浅い接続で終わらせず、 一気通貫で価値まで届ける。

04

Brain First / Design First

GBrainで知識を蓄積し、設計書→アーキテクチャ図→ 論理矛盾チェック→実装の順を厳守。 「コードを書く前に設計する」を鉄則とする。 形式検証（Alloy）で状態機械の矛盾を証明してから 実装に入る。
